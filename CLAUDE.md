## Project Overview

A Leaflet.js GIS web application for "Jaktlaget" — a hunting field map for Sund Jaktfelt in Sund, Nordland, Norway. Fully static frontend, no build step. Production is deployed to GitHub Pages via GitHub Actions. All data is stored in Supabase. Authentication is handled via Supabase Auth.

## Running the App

**Local development:** Open `public/index.html` via VS Code Live Server. Ensure `public/config.js` exists locally with your Supabase credentials (gitignored — see Credentials section below).

**Production:** Deployed automatically to `https://andreaszahl.github.io/jaktlaget` on every push to `main` via `.github/workflows/deploy.yml`.

## Architecture

### Frontend (`public/`)
- **`login.html`** — login page: dark forest green background with topographic SVG overlay, centered white card with Supabase Auth sign-in. Redirects to `index.html` on success or if already authenticated.
- **`index.html`** — main map page: header bar (app name, user greeting, logout), fixed 190px sidebar (Kartlag + Navigasjon sections), Leaflet map with Kartverket Topo basemap, weather panel, draggable Poster markers, read-only Fellinger markers.
- **`statistikk.html`** — statistics page: same sidebar design, filterable table of `fellinger` data.
- **`auth.js`** — shared auth guard: `requireAuth()` checks Supabase session and redirects to `login.html` if not authenticated. Called at the top of every protected page.
- **`weatherService.js`** — fetches Met.no Locationforecast 2.0 API, caches 5 minutes, converts wind degrees to compass/arrow.
- **`supabaseClient.js`** — initializes Supabase JS client (`db`) from `window.APP_CONFIG`; gitignored.

### Map layers
| Layer | Source | Notes |
|---|---|---|
| Kartverket Topo | WMTS tile layer | Default basemap |
| Poster | Supabase `markers` table | Draggable, numbered icons, saves on drop, Ctrl+Z undo |
| Fellinger | Supabase `fellinger` table | Read-only, off by default in layer control |

### Supabase tables
| Table | Columns | Used by |
|---|---|---|
| `markers` | `id, name, lat, lon` | Poster layer in `index.html` |
| `fellinger` | `id, beskrivelse, year, latitude, longitude` | Fellinger layer in `index.html`, `statistikk.html` |

Both tables have Row Level Security enabled — policies require `auth.role() = 'authenticated'`.

### Auth flow
1. Unauthenticated users are redirected to `login.html` by `requireAuth()` in `auth.js`.
2. `login.html` calls `db.auth.signInWithPassword` and redirects to `index.html` on success.
3. `index.html` shows the logged-in user's email via `db.auth.getSession()` and provides a logout button (`db.auth.signOut` → `login.html`).

### Credentials / config
- `public/config.js` — gitignored; create locally with `window.APP_CONFIG = { SUPABASE_URL, SUPABASE_ANON_KEY }`
- Generated at deploy time by GitHub Actions from `SUPABASE_URL` and `SUPABASE_ANON_KEY` repository secrets
- `supabaseClient.js` reads credentials from `window.APP_CONFIG`

### Deployment
`.github/workflows/deploy.yml` runs on push to `main`:
1. Generates `public/config.js` from GitHub Secrets
2. Deploys `public/` to GitHub Pages
